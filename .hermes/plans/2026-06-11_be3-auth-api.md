# backend#3 인증 API 구현 계획

## 목표

소셜 OAuth 검증 결과를 기반으로 기존 유저 로그인, 초대 코드 가입, JWT access/refresh token 발급/검증, GraphQL 인증 컨텍스트 주입 기반을 구현한다.

## 범위

- `AuthServiceModule` 구성
- `AuthService`: 로그인, 초대 코드 가입, 토큰 갱신
- `JwtService`: HS256 JWT 서명/검증, access/refresh token 발급
- `SocialOAuthVerifier`: provider별 OAuth 토큰 검증 추상화
- `AuthResolver`: `login`, `joinWithInviteCode`, `refreshToken`, `me`
- `AuthGuard`: GraphQL context의 `currentUser` 기반 보호
- TDD 유닛 테스트

## 범위 제외

- 실제 Kakao/Google/Apple 외부 API 연동은 adapter 교체 지점만 만든다.
- SMS 휴대폰 인증은 별도 후속 이슈로 남긴다.
- refresh token 저장소/폐기 목록은 DB 모델이 없으므로 stateless JWT 검증으로 처리한다.

## 구현 순서

1. Red: `AuthService` 유닛 테스트 작성
   - 기존 유저 로그인 성공
   - 미가입 이메일 로그인 실패
   - 유효 초대 코드 가입 성공 및 사용 횟수 증가
   - 중복 이메일 가입 실패
   - refresh token으로 새 token pair 발급
   - 변조/만료 토큰 실패
2. Green: Auth service/JWT/verifier 최소 구현
3. Resolver/Module 연결 및 codegen
4. Verify: `npm test -- --runInBand`, `npm run lint`, `npm run build`

## 보안 기준

- JWT secret은 `JWT_SECRET` 환경변수를 우선 사용하고 테스트/개발 기본값은 명시적으로 fallback한다.
- OAuth provider는 `KAKAO`, `GOOGLE`, `APPLE`만 허용한다.
- 초대 코드 가입 시 회사 정원, 만료, 비활성, 사용 한도를 검증한다.
